### 解释：对问题进行规范化解释

**Transformer** 是现代大语言模型最核心的基础架构。它最重要的突破，是用 **自注意力机制（Self-Attention）** 替代了传统 RNN 那种“按时间步串行处理”的方式，让模型能够在一个序列内部直接建模任意两个 token 之间的关系。

对大模型来说，你可以把 Transformer 理解成一个“反复做两件事”的堆叠结构：

1. **看上下文关系**：通过 Attention 判断“当前 token 应该关注前面哪些 token”
2. **做非线性变换**：通过 FFN/MLP 提升表示能力

之所以它成为 LLM 的标准底座，是因为它同时具备：

* 很强的并行训练能力
* 很好的长距离依赖建模能力
* 易于扩展到超大参数量

### 举例：举例说明问题，帮助理解核心机制

假设一句话是：

`The animal didn't cross the street because it was too tired.`

当模型处理 `it` 时，关键问题是：`it` 指的是谁？

自注意力会让 `it` 对前面的多个 token 计算相关性分数：

* `animal`
* `cross`
* `street`
* `tired`

最后模型通常会发现，`it` 和 `animal` 的语义关联更强，于是会更关注 `animal` 对当前位置的表示贡献。

也就是说，**Attention 的本质就是“动态加权取上下文信息”**。

### 知识点：着重说明必须记住的技术点

1. **Transformer 的基本结构**

   * 输入嵌入（Embedding）
   * 位置编码（Position Encoding / RoPE）
   * 多头自注意力（Multi-Head Attention）
   * 前馈网络（FFN / MLP）
   * 残差连接（Residual）
   * 归一化（LayerNorm / RMSNorm）

2. **Q、K、V 的含义**

   * **Q（Query）**：当前位置“想找什么信息”
   * **K（Key）**：每个位置“我能提供什么信息”
   * **V（Value）**：真正被聚合输出的内容
   * 注意力权重通常来自 `Q * K^T`，再经过缩放和 softmax

3. **缩放点积注意力**

   * 标准公式可以记成：
   * `Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V`
   * 除以 `sqrt(d_k)` 是为了防止维度大时点积值过大，导致 softmax 过于尖锐、训练不稳定

4. **多头注意力为什么有效**

   * 单个头只能学一种关系模式
   * 多头允许模型在不同子空间同时关注不同关系
   * 例如一个头关注语法依赖，另一个头关注指代关系，还有的头关注位置模式

5. **Encoder-only、Decoder-only、Encoder-Decoder**

   * **Encoder-only**：适合理解任务，如 BERT
   * **Decoder-only**：适合生成任务，如 GPT、LLaMA、Qwen
   * **Encoder-Decoder**：适合条件生成，如 T5、BART
   * 当前大多数通用大模型属于 **Decoder-only**

6. **因果掩码（Causal Mask）**

   * 自回归生成时，当前位置不能偷看未来 token
   * 所以在 attention 矩阵上要屏蔽右上角未生成部分
   * 这保证训练目标和推理过程一致

7. **复杂度要记住**

   * 标准 self-attention 对序列长度 `n` 的时间/显存复杂度通常是 `O(n^2)`
   * 这也是长上下文场景的核心瓶颈之一
