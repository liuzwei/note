# 监督微调

在[第 2 章第 2 节](https://huggingface.co/learn/llm-course/chapter2/2)中，我们已经看到，生成式语言模型可以针对特定任务进行微调，比如摘要生成和问答。不过在今天，更常见的做法是让语言模型同时在更广泛的一组任务上进行微调，这种方法叫做**监督微调（Supervised Fine-Tuning，SFT）**。

这个过程能够帮助模型变得更加通用，并更好地处理多样化的使用场景。大多数人们在 ChatGPT 这类平台上接触到的大语言模型，都经过了 SFT，以便让它们更有帮助，也更符合人类偏好。

本章分为四个部分：

## 1. Chat Templates

Chat 模板用于规范用户与 AI 模型之间的交互结构，确保模型能够以一致、符合上下文的方式响应。它通常包含系统提示词、基于角色的消息结构等组成部分。

## 2. 监督微调

监督微调（SFT）是把预训练语言模型适配到具体任务上的关键步骤。它的核心做法是：使用带标签的任务数据集继续训练模型。

如果你想看更完整的操作指南，包括关键步骤和最佳实践，可以参考：

- <https://huggingface.co/docs/trl/en/sft_trainer>

## 3. 低秩适配（LoRA）

低秩适配（LoRA）是一种微调语言模型的技术，它通过向模型层中加入低秩矩阵来完成高效微调。

这种方法的一个核心优点是：在保留预训练模型知识的同时，显著减少训练时所需的显存与可训练参数量，因此即使在资源有限的硬件上，也可以对较大的模型进行微调。

## 4. 评估

评估是微调流程中不可缺少的一步。它帮助我们衡量模型在特定任务数据集上的表现。
