# 记忆（Memory）

> 让 Agent 在对话之间记住偏好、上下文和知识

记忆（Memory）使 Agent 能够在对话之间记住信息。与每次从空白状态开始的传统聊天机器人不同，具有记忆的 Agent 可以建立持久的知识库——用户的编码风格、项目约定、常见问题的首选解决方案——这些信息在每次新对话中都会加载。

记忆作为 `AGENTS.md` 文件存储在[后端（Backend）](/oss/python/deepagents/backends)中。Agent 在启动时读取这些文件并将其注入到系统提示中，或按需读取。

## 记忆如何工作（How memory works）

1. **将 Agent 指向记忆文件。** 在创建 Agent 时将文件路径传递给 `memory=`。您也可以通过 `skills=` 传递[技能（Skills）](/oss/python/deepagents/skills)用于程序性记忆（可重用的指令，告诉 Agent *如何*执行任务）。[后端（Backend）](/oss/python/deepagents/backends)控制文件存储位置以及谁可以访问它们。
2. **Agent 读取记忆。** Agent 可以在启动时将记忆文件加载到系统提示中，或者在对话期间按需读取。例如，[技能（Skills）](/oss/python/deepagents/skills)使用按需加载：Agent 在启动时仅读取技能描述，然后在匹配到任务时才读取完整的技能文件。这保持了上下文的精简，直到需要某项能力时才加载。
3. **Agent 更新记忆（可选）。** 当 Agent 学到新信息时，它可以使用内置的 `edit_file` 工具更新记忆文件。更新可以在对话期间进行（默认），也可以在对话之间通过[后台整理（background consolidation）](#background-consolidation)进行。更改会被持久化并在下一次对话中可用。并非所有记忆都是可写的：开发者定义的[技能（Skills）](/oss/python/deepagents/skills)和[组织级策略](#organization-level-memory)通常是只读的。详见[只读与可写记忆](#read-only-vs-writable-memory)。

最常见的两种模式是[Agent 级记忆](#agent-scoped-memory)（在所有用户间共享）和[用户级记忆](#user-scoped-memory)（每个用户隔离）。

## 作用域记忆（Scoped memory）

Agent 记忆可以设置作用域，使相同的记忆文件对使用 Agent 的所有人都可访问，或者使记忆文件对每个用户独立。

### Agent 级记忆（Agent-scoped memory）

赋予 Agent 自己的持久身份，随时间推移而演进。Agent 级记忆在所有用户间共享，因此 Agent 通过每一次对话建立自己的个性、积累知识和习得的偏好。在与用户交互的过程中，它发展专业知识、优化方法并记住什么有效。当具有写入权限时，它还可以学习和更新[技能（Skills）](/oss/python/deepagents/skills)。

关键在于后端命名空间：将其设置为 `(assistant_id,)` 意味着此 Agent 的每次对话都读取和写入同一个记忆文件。

<Note>
  访问 `rt.server_info` 需要 `deepagents>=0.5.0`。在旧版本上，改为从 `get_config()["metadata"]["assistant_id"]` 读取助手 ID。
</Note>

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    memory=["/memories/AGENTS.md"],
    skills=["/skills/"],
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(
                namespace=lambda rt: (
                    rt.server_info.assistant_id,  # [!code highlight]
                ),
            ),
            "/skills/": StoreBackend(
                namespace=lambda rt: (
                    rt.server_info.assistant_id,  # [!code highlight]
                ),
            ),
        },
    ),
)
```

  config2 = {"configurable": {"thread_id": str(uuid7())}}

      config=config2,
  )

### 用户级记忆（User-scoped memory）

为每个用户提供他们自己的记忆文件。Agent 按用户记住偏好、上下文和历史，而核心 Agent 指令保持不变。当多个用户共享同一个 Agent 部署时，用户可以根据自己的喜好获得个性化体验。

关键是将记忆文件隔离到按用户身份键控的命名空间中：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    memory=["/memories/AGENTS.md"],
    skills=["/skills/"],
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(
                namespace=lambda rt: (
                    rt.server_info.user.identity,  # [!code highlight]
                ),
            ),
            "/skills/": StoreBackend(
                namespace=lambda rt: (
                    rt.server_info.user.identity,  # [!code highlight]
                ),
            ),
        },
    ),
)
```

## 高级用法（Advanced usage）

### 情节记忆（Episodic memory）

使用 `langgraph.store` 搜索先前的对话以检索相关上下文。这与程序性记忆（技能和偏好）不同——情节记忆允许 Agent 回忆*特定过去事件*，如用户之前报告的 bug 或上周讨论的设计决策。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from langgraph.store import SearchItem

# Search conversations for a specific user
results: list[SearchItem] = store.search(
    ("conversations", "user-123"),
    query="What did we decide about the authentication flow?",
)
```

`store.search()` 返回一个 `SearchItem` 列表，每个包含 `value`（存储的值）、`key`（路径）、`created_at` 和 `updated_at` 字段。您可以使用这些来检索特定线程的内容。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# Search conversations across an organization
results = store.search(
    ("conversations", "org-456"),
    query="database migration plan",
    limit=5,
)
```

### 组织级记忆（Organization-level memory）

您可以在多个 Agent 之间共享记忆文件。每个 Agent 可以看到由组织管理的知识库。组织级记忆通常是只读的——Organization 拥有内容，Agent 消费内容。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    memory=["/policies/code-style.md", "/policies/deployment.md"],
    backend=CompositeBackend(
        default=StateBackend(),
        routes={
            "/policies/": StoreBackend(
                namespace=lambda rt: (rt.context.org_id, "policies"),
            ),
        },
    ),
    permissions=[
        FilesystemPermission(
            operations=["write"],
            paths=["/policies/**"],
            mode="deny",  # Agent can read but not modify org policies
        ),
    ],
)
```

组织级记忆与技能（Skills）和偏好/记忆配合使用。组织级记忆存储全局策略和指南（您*必须*遵守的规则），而偏好是 Agent 从交互中学到的（它*倾向于*做的），技能是程序性知识（它*能够*做的）。

### 后台整理（Background consolidation）

后台整理（Background consolidation）使 Agent 能够在后台处理记忆，独立于主动会话。它对于维护会随着 Agent 使用而增长的长期记忆很有用。

有两种方式实现后台整理：使用**整理 Agent**（Consolidation Agent），或使用 **Cron** 作业。

#### 整理 Agent（Consolidation agent）

整理 Agent（Consolidation Agent）是以部署为目标的 Agent——一个专用的 Agent 进程，被调用以处理其他 Agent 的记忆。它定期运行，查看它应该整理的 Agent 的记忆，通过合并、摘要或清理来更新它们。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
consolidation_prompt = """You are a memory consolidation specialist.

Review the conversation transcripts and update the shared memory file to reflect
what the agent has learned. The memory file contains the current preferences,
knowledge, and patterns. Your job is to:

1. Extract new preferences, patterns, and knowledge from conversations
2. Update the memory file - consolidate related information, remove outdated entries
3. Never delete important information, only consolidate"""

consolidation_agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    system_prompt=consolidation_prompt,
    memory=["/memories/AGENTS.md"],
    backend=StoreBackend(
        namespace=lambda rt: (
            rt.server_info.assistant_id,
        ),
    ),
)
```

#### Cron

使用 cron 作业或调度程序按固定间隔运行整理 Agent。这对于需要持续维护知识的长期运行的 Agent 特别有用。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
import time
from datetime import datetime, timezone

def consolidation_loop():
    while True:
        now = datetime.now(timezone.utc)

        # Get recent conversations from the store
        recent = store.search(
            ("conversations", assistant_id),
            query="",
            limit=50,
        )

        # Run consolidation
        result = consolidation_agent.invoke({
            "messages": [{
                "role": "user",
                "content": f"Review and consolidate memory from {len(recent)} conversations.",
            }],
        })

        # Wait before next consolidation
        time.sleep(3600)  # Run every hour
```

### 只读与可写记忆（Read-only vs writable memory）

默认情况下，记忆文件是可写的——Agent 可以读取和修改它们。这对于 Agent 学到的偏好和知识很有用。

要使记忆只读，请使用权限（Permissions）来阻止对特定路径的写入。这对于组织级策略或不应由 Agent 修改的共享知识库很有用。参见[只读记忆示例](#read-only-memory)。

### 考虑事项

#### 人工介入验证（Human-in-the-loop validation）

在 Agent 写入共享记忆之前添加人工介入验证。使用中断机制要求对写入敏感路径进行人工审批。

要强制执行只读记忆，使用权限声明式地拒绝写入特定路径。对于自定义验证逻辑（速率限制、审计日志记录、内容检查），使用后端策略钩子（backend policy hooks）。

#### 并发写入（Concurrent writes）

多个线程可以并行写入记忆，但并发写入同一文件可能导致最后写入者获胜的冲突。对于用户级记忆，这种情况很少见，因为用户通常一次只有一个活跃的对话。对于 Agent 级或组织级记忆，考虑使用后台整理来序列化写入，或将记忆结构化为按主题分隔的多个文件以减少争用。

在实践中，如果写入因冲突而失败，LLM 通常足够智能以重试或优雅地恢复，因此单次写入丢失不会造成灾难性后果。

#### 同一部署中的多个 Agent（Multiple agents in the same deployment）

要为共享部署中的每个 Agent 提供自己的记忆，在命名空间中添加 `assistant_id`：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
StoreBackend(
    namespace=lambda rt: (
        rt.server_info.assistant_id,
        rt.server_info.user.identity,
    ),
)
```

如果只需要 Agent 级隔离而不需要按用户作用域，单独使用 `assistant_id`。

使用 LangSmith 追踪来审计您的 Agent 写入记忆的内容。每次文件写入在追踪中都显示为工具调用。
