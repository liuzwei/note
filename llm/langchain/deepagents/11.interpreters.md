# 解释器（Interpreters）

> 在安全的进程内运行时中执行代码

解释器（Interpreters）为在安全的进程中运行代码提供了一个轻量级的执行环境。与提供完整 OS 级隔离的沙箱不同，解释器在嵌入式 QuickJS 上下文中运行 JavaScript/TypeScript——启动时间极短且资源消耗低。

## 何时使用解释器（When to use an interpreter）

| 场景 | 解释器 | 沙箱 |
|---------|-------------|---------|
| **启动时间** | 毫秒级 | 秒级 |
| **隔离** | 进程内，内存隔离 | 完整 OS 级隔离 |
| **语言** | JavaScript / TypeScript | 任何语言 |
| **文件系统** | 虚拟内存文件系统 | 真实或虚拟文件系统 |
| **网络** | 仅 PTC（工具调用） | 取决于配置 |
| **包安装** | 否 | 是 |
| **最佳用途** | 快速计算、数据转换、数学运算 | 完整代码执行、测试、系统交互 |

解释器非常适合：

- 数据格式化和转换
- 数学计算和统计
- 在合成步骤之前验证结构化输出
- 快速原型和算法探索

## 选择正确的执行路径（Choose the right execution path）

| 需求 | 推荐 |
|--------|-----------------|
| 快速计算（数学、数据转换） | 解释器 |
| 运行 shell 命令或安装包 | 沙箱 |
| 两者都需要 | 两者结合——使用[复合后端（CompositeBackend）](/oss/python/deepagents/backends#compositebackend) |

## 向 Agent 添加解释器（Add an interpreter to an agent）

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import create_deep_agent
from deepagents.backends import StateBackend
from langchain_quickjs import CodeInterpreterMiddleware

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=StateBackend(),
    middleware=[CodeInterpreterMiddleware()],
)
```

## 在解释器中运行代码（Run code in the interpreter）

中间件向 Agent 添加一个 `eval` 工具。该工具在持久化上下文中运行 TypeScript，捕获 `console.log`，并返回最后一个表达式的结果。

Agent 可以编写如下代码：

```javascript theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
const rows = [
  { team: "alpha", score: 8 },
  { team: "beta", score: 13 },
  { team: "alpha", score: 21 },
];

const totals = rows.reduce((acc, row) => {
  acc[row.team] = (acc[row.team] ?? 0) + row.score;
  console.log(`${row.team} score: ${acc[row.team]}`);
  return acc;
}, {} as Record<string, number>);

totals;
// Output:
// alpha score: 8
// beta score: 13
// alpha score: 29
// Result: { alpha: 29, beta: 13 }
```

解释器上下文在 Agent 轮次之间保持持久化，通过快照功能实现增量状态。变量、函数和导入在 `eval` 调用之间保持可用。Agent 可以逐步构建——导入库、加载数据、然后对数据进行操作。

<Warning>
  **重要**

  解释器在进程内运行。长时间运行或无限循环将阻塞 Agent。使用 `timeout` 选项设置每个 eval 调用的时间限制。
</Warning>

## PTC（编程式工具调用）

PTC（Programmatic Tool Calls，编程式工具调用）允许解释器代码直接调用其他 Agent 工具。这使得数据流模式成为可能，例如"处理数据，然后对结果调用 `write_file`"。

```javascript theme={"theme":{"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}}
// Inside the interpreter
const input = await tools.read_file({ path: "/data/input.json" });
const parsed = JSON.parse(input);
const result = transformed_data(parsed);  // pure JS transform
await tools.write_file({ path: "/data/output.json", content: JSON.stringify(result) });
```

<Warning>
  **重要**

  PTC 调用当前通过解释器桥接执行，不经过正常的工具调用路径。因此，`interrupt_on` 审批工作流不会针对每个 PTC 调用的工具调用强制执行。
</Warning>

### 限制 PTC 访问

默认情况下，`tools.*` 对象暴露所有 Agent 工具。使用 `ptc` 参数限制可用的工具：

```python theme={"theme":{"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}}
from deepagents.tools import ReadFileTool, WriteFileTool

CodeInterpreterMiddleware(
    ptc=[ReadFileTool, WriteFileTool],  # Only read_file and write_file
)
```

## 递归语言模型（Recursive language models）

解释器可以调用 LLM 工具来生成文本、分析数据或做出决策：

```javascript theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
const sentiment = await tools.llm({
  messages: [
    { role: "system", content: "Classify sentiment as positive, negative, or neutral." },
    { role: "user", content: reviewText },
  ],
  model: "google_genai:gemini-3.1-pro-preview",
});
```

这使解释器能够处理传统上需要外部 API 调用的任务——文本分类、实体提取、摘要——所有这些都在同一个解释器上下文中完成。

## 解释器技能（Interpreter skills）

技能（Skills）可以包含 `module` 字段，引用在解释器上下文中执行的脚本文件。Agent 可以在需要时导入和运行这些模块。参见[使用解释器技能](/oss/python/deepagents/skills#use-interpreter-skills)。

<Note>
  解释器技能模块通过解释器的 `skills_backend` 解析。Agent 发现技能并在首次需要调用其模块时加载模块代码。
</Note>

## 快照和时间旅行（Snapshots and time travel）

解释器支持状态快照，允许在轮次之间持久化其上下文。启用后，解释器状态在人类介入中断或检查点恢复后仍然保持。

```python theme={"theme":{"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}}
CodeInterpreterMiddleware(
    snapshot_between_turns=True,  # Default: True
    max_snapshot_bytes=1024 * 1024,  # 1 MB max snapshot size
)
```

快照捕获：

- 所有 JavaScript 变量和函数
- PTC 工具引用（当被顶层代码拉取时）
- 控制台输出缓冲区

## 安全性和限制（Security and limits）

| 能力 | 默认 | 如何启用 |
|-----------|---------|-----------------|
| 解释器技能模块 | 否 | 添加模块条目并配置 `skills_backend` |
| 文件系统访问 | 否 | 通过 PTC 允许列表添加内置文件系统工具 |
| 网络访问 | 否 | 通过 PTC 暴露特定的网络工具 |
| 时钟/日期时间访问 | 否 | 如果需要，暴露显式的时间工具 |
| Shell 命令、包安装、测试、OS 级执行 | 否 | 使用沙箱后端 |

**代码执行方式**：解释器代码在嵌入式 QuickJS 上下文中运行，而不是在单独的 VM 或进程中。在 Python 中，此运行时由 `quickjs-rs` 提供，其在[安全指南](https://github.com/quickjs-rs/quickjs-rs?tab=readme-ov-file#security)中记录了同进程执行边界。将解释器视为能力作用域的执行层，而不是主机内存隔离边界。对于不受信任或半受信任的代码，在隔离的工作进程或容器中运行 Agent，并保持 PTC 允许列表狭窄。

## 中间件选项（Middleware options）

`CodeInterpreterMiddleware` 接受以下选项：

| 参数 | 默认值 | 用途 |
|--------|---------|-------|
| `memory_limit` | 64 * 1024 * 1024（64 MB） | QuickJS 堆内存限制（字节）。 |
| `timeout` | 5.0 | 每次 eval 的超时时间（秒）。 |
| `max_ptc_calls` | 256 | 每次 eval 的最大 `tools.*` 调用次数。仅在受信任环境中使用 `None`。 |
| `tool_name` | "eval" | 暴露给模型的解释器工具名称。 |
| `max_result_chars` | 4000 | 从 `result` 和 `stdout` 块返回的最大字符数。 |
| `capture_console` | True | 是否捕获 `console.log`、`console.warn` 和 `console.error` 输出。 |
| `ptc` | None | PTC 允许列表：工具名称或 `BaseTool` 实例的列表。 |
| `skills_backend` | None | 用于解析解释器技能模块的后端。 |
| `snapshot_between_turns` | True | 解释器状态快照是否在 Agent 轮次之间持久化。 |
| `max_snapshot_bytes` | None | 最大序列化快照大小。默认为 `memory_limit`。 |
