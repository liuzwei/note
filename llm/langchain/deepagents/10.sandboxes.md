# 沙箱（Sandboxes）

> 在隔离环境中安全执行代码

沙箱（Sandboxes）为在隔离环境中运行代码提供了安全、受控的执行上下文。它们与虚拟文件系统配对，允许 Agent 读写文件，同时对主机系统零风险。

## 为什么使用沙箱？（Why use sandboxes?）

| 场景 | 无沙箱 | 有沙箱 |
| -------- | ----------- | ------- |
| **代码执行** | 直接在主机上运行：安全风险 | 在隔离环境中运行 |
| **文件访问** | Agent 可以访问主机上的任何内容 | 文件包含在沙箱虚拟文件系统中 |
| **网络** | 无限制的网络访问 | 可以限制或阻止 |
| **系统调用** | 完全系统访问 | 由沙箱配置控制 |
| **持久性** | 在主机上留下文件 | 沙箱可以在使用后丢弃 |

<Note>
  如果您不需要沙箱提供的全部执行能力——只需要在安全进程中运行快速计算——请考虑使用[解释器（Interpreter）](/oss/python/deepagents/interpreters)。解释器在嵌入式 QuickJS 上下文中运行代码，启动速度快且资源消耗低。沙箱提供完整的操作系统级隔离，但启动成本更高。
</Note>

## 基本用法（Basic usage）

使用 E2B 沙箱：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import create_deep_agent
from langchain_e2b import E2BSandbox

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=E2BSandbox(),
    system_prompt="You are a coding assistant with sandbox access. You can create and run code in the sandbox.",
)
```

然后 Agent 可以使用 `execute` 工具在沙箱中运行命令：

```python
agent.invoke({
    "messages": [{"role": "user", "content": "Create a Python script that generates a fibonacci sequence"}],
})
```

Agent 将创建文件并在沙箱中执行它们，然后返回结果。

使用 Daytona 沙箱：

```python
from deepagents import create_deep_agent
from langchain_daytona import DaytonaSandbox
from daytona import Daytona, DaytonaConfig

daytona = Daytona(DaytonaConfig(
    api_key="<your-api-key>",
    server_url="<your-server-url>",
    target="us",
))
sandbox = daytona.create()

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=DaytonaSandbox(sandbox=sandbox),
    system_prompt="You are a coding assistant with sandbox access.",
)

try:
    result = agent.invoke(...)
finally:
    sandbox.stop()  # Clean up
```

使用 Modal 沙箱：

```python
from deepagents import create_deep_agent
from langchain_modal import ModalSandbox

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=ModalSandbox(),
    system_prompt="You are a coding assistant with sandbox access.",
)
```

使用 AgentCore 沙箱：

```python
from deepagents import create_deep_agent
from deepagents.backends import AgentCoreSandbox

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=AgentCoreSandbox(),
    system_prompt="You are a coding assistant with sandbox access.",
)
```

## 可用提供商（Available providers）

| 提供商 | SDK 包 | 后端类 |
| -------- | --------- | ------------ |
| E2B | `langchain-e2b` | `E2BSandbox` |
| Daytona | `langchain-daytona` | `DaytonaSandbox` |
| Modal | `langchain-modal` | `ModalSandbox` |
| AgentCore | `deepagents`（内置） | `AgentCoreSandbox` |

所有提供商都实现相同的虚拟文件系统接口，因此代码在提供商之间可以移植。

## 生命周期和作用域（Lifecycle and scoping）

沙箱可以按线程或按助手进行作用域管理。

### 线程作用域（默认）

默认情况下，沙箱与线程生命周期绑定。当线程结束时，沙箱会被清理。这确保了每个对话从干净状态开始。

```python
agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=E2BSandbox(),
)

# Thread 1: gets its own sandbox
result1 = agent.invoke(
    {"messages": [{"role": "user", "content": "Create a script"}]},
    config={"configurable": {"thread_id": "thread-1"}},
)

# Thread 2: gets a different sandbox
result2 = agent.invoke(
    {"messages": [{"role": "user", "content": "Create another script"}]},
    config={"configurable": {"thread_id": "thread-2"}},
)
```

### 助手作用域（Assistant-scoped）

要在同一线程的多次调用之间共享沙箱，请使用助手作用域。这对于长时间运行的开发任务或需要在交互之间保持状态的场景很有用。

```python
from deepagents import create_deep_agent
from langchain_e2b import E2BSandbox

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=E2BSandbox(scoping="assistant"),
)

# First invocation
result1 = agent.invoke(
    {"messages": [{"role": "user", "content": "Install pandas and numpy"}]},
    config={"configurable": {"thread_id": "session-1"}},
)

# Second invocation - same sandbox, packages still available
result2 = agent.invoke(
    {"messages": [{"role": "user", "content": "Now use them to analyze this data"}]},
    config={"configurable": {"thread_id": "session-1"}},
)
```

### 沙箱作为工具模式（Sandbox as tool pattern）

主 Agent 运行在常规后端上，但有一个专门的"沙箱"工具用于代码执行。这对于主要执行文本操作但偶尔需要运行代码的 Agent 很有用。

```python
# Can also do this with AgentCore, E2B, Runloop, Modal
sandbox = Daytona().create()
backend = DaytonaSandbox(sandbox=sandbox)

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    backend=backend,
    system_prompt="You are a coding assistant with sandbox access. You can create and run code in the sandbox.",
)
```

## 使用文件（Working with files）

沙箱后端实现了与其他 Deep Agents 后端相同的虚拟文件系统接口。所有标准的文件系统工具——`read_file`、`write_file`、`edit_file`、`ls`、`glob`、`grep`——都可以在沙箱中使用，使用沙箱的虚拟文件系统而不是主机文件系统。

```python
# These all work the same way in sandbox backends
agent.invoke({
    "messages": [
        {"role": "user", "content": """
        1. Write a Python script to /home/user/analysis.py
        2. Run it with: python /home/user/analysis.py
        3. Read the output
        """}
    ],
})
```

### 为沙箱提供初始数据（Seeding the sandbox）

您可以通过以下多种方式将数据放入沙箱：

1. **使用输入文件（Input files）**：在调用时传递初始数据：

```python
from deepagents.backends.utils import create_file_data

agent.invoke({
    "messages": [{"role": "user", "content": "Analyze this data"}],
    "files": {
        "/home/user/data.csv": create_file_data("col1,col2\n1,2\n3,4"),
    },
})
```

2. **从 Agent 写入**：Agent 可以创建文件，然后读取或执行它们。
3. **使用技能（Skills）**：技能可以包括由沙箱执行的脚本。

### 检索制品（Retrieving artifacts）

当 Agent 在沙箱中创建文件时，这些文件会保留在沙箱的虚拟文件系统中。您可以在沙箱创建后从其中读取文件：

```python
from deepagents.backends import SandboxBackendProtocol

backend: SandboxBackendProtocol = agent.backend
artifacts = backend.list_files("/home/user/output/")
```

<Warning>
  沙箱后端之间的路径行为不同。始终检查您的提供商的文档以了解特定的文件系统路径和行为。
</Warning>

## 安全考虑（Security considerations）

### 安全处理密钥（Handling secrets safely）

<Warning>
  **不要直接将密钥注入沙箱。** 沙箱内的代码可能会通过网络泄漏密钥，绕过标准保护措施。
</Warning>

如果您必须将密钥注入沙箱（不推荐），请采取以下预防措施：

- 启用对所有工具调用的人工介入审批，而不仅仅是敏感调用
- 阻止或限制沙箱的网络访问以限制泄露路径
- 使用尽可能窄的凭证范围和最短的生命周期
- 监控沙箱网络流量中意外的出站请求

即使有这些防护措施，这仍然是一个不安全的变通方案。足够巧妙的上下文注入攻击可以绕过输出过滤和 HITL 审查。

### 通用最佳实践（General best practices）

- 在应用程序中信任沙箱输出之前先进行审查
- 在不需要时阻止沙箱网络访问
- 使用中间件过滤或编辑工具输出中的敏感模式
- 将沙箱内产生的所有内容视为不受信任的输入
