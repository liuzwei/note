# 配置文件（Profiles）

> 封装 Deep Agents 在模型选定后应用的各提供商和各模型的默认配置

**操控配置文件（Harness profiles）** 让您可以封装 Deep Agents 在选定某个提供商或特定模型时应用的配置：系统提示词调整、工具描述覆盖、排除的工具或中间件、额外的中间件以及通用子 Agent 编辑。它们是在不修改 `create_deep_agent` 调用点的情况下，调整操控（Harness）对特定模型行为的主要方式。在 Python 中构建配置文件时使用 `HarnessProfile`；[加载或保存 YAML/JSON 文件](#load-profiles-from-config-files)时使用 `HarnessProfileConfig`。Deep Agents 为 OpenAI 和 Anthropic（Claude）模型内置了操控配置文件。

**提供商配置文件（Provider profiles）** 是一个更窄的配套 API，用于*模型构造*参数，不影响操控。大多数调用者不需要它们；当您想要通过提供商选择设置 `init_chat_model` 默认值、凭据检查或运行时派生的参数作为默认值时，可以使用它们（例如，在打包提供商集成时）。

## 操控配置文件（Harness profiles）

`HarnessProfile` 描述了 `create_deep_agent` 在构造聊天模型后应用的提示词组装、工具可见性、中间件和默认子 Agent 调整：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import (
    GeneralPurposeSubagentProfile,
    HarnessProfile,
    register_harness_profile,
)

register_harness_profile(
    "openai:gpt-5.4",
    HarnessProfile(
        system_prompt_suffix="Respond in under 100 words.",
        excluded_tools={"execute"},
        excluded_middleware={"SummarizationMiddleware"},
        general_purpose_subagent=GeneralPurposeSubagentProfile(enabled=False),
    ),
)
```

<ResponseField name="base_system_prompt" type="string">
  替换 Deep Agents 的基础系统提示（[提示词组装](/oss/python/deepagents/customization#prompt-assembly)中的 `CUSTOM`）。
</ResponseField>

<ResponseField name="system_prompt_suffix" type="string">
  在组装的提示词末尾追加文本（[提示词组装](/oss/python/deepagents/customization#prompt-assembly)中的 `SUFFIX`）；应用于主 Agent、声明性子 Agent 和自动添加的通用子 Agent。
</ResponseField>

<ResponseField name="tool_description_overrides" type="Mapping[str, str]">
  按工具名称键覆盖单个工具描述。
</ResponseField>

<ResponseField name="excluded_tools" type="frozenset[str]">
  从工具集中移除特定的操控层工具。按工具名称（字符串）匹配，作为注入后过滤器应用，因此可以删除用户提供的工具以及操控中间件添加的工具。参见[在没有默认文件系统工具的情况下运行](/oss/python/deepagents/harness#virtual-filesystem-access)的完整示例。
</ResponseField>

<ResponseField name="excluded_middleware" type="frozenset[type[AgentMiddleware] | str]">
  从栈中移除特定的中间件类。接受中间件类或字符串名称。
</ResponseField>

<ResponseField name="extra_middleware" type="Sequence[AgentMiddleware] | Callable[[], Sequence[AgentMiddleware]]">
  将中间件追加到此配置文件应用的每个栈。
</ResponseField>

<ResponseField name="general_purpose_subagent" type="GeneralPurposeSubagentProfile">
  禁用、重命名或修改通用子 Agent 的提示词。当此字段的 `system_prompt` 与 `base_system_prompt` 同时设置时，通用子 Agent 特定的提示词优先——参见[通用子 Agent 提示词](/oss/python/deepagents/customization#general-purpose-subagent-prompt)。
</ResponseField>

<Note>
  调用者提供的 `system_prompt=` 始终位于组装提示词的开头，而 `system_prompt_suffix` 始终位于末尾——无论选定哪个模型。相同的覆盖规则适用于子 Agent：每个子 Agent 自己的 `system_prompt=` 在其提示词的开头，而 `system_prompt_suffix` 在末尾。
</Note>

### 配置文件匹配逻辑

注册配置文件时，Deep Agents 通过以下逻辑将当前选定的模型匹配到配置文件：

1. **精确匹配**：首先检查 `<provider>:<model>`（例如 `"openai:gpt-5.4"`）。
2. **提供商回退**：如果未找到，回退到 `<provider>` 密钥（例如 `"openai"`）。
3. **默认值**：如果仍未找到，应用最外层 `create_deep_agent` 的值。

这使您可以对大多数模型使用广泛的提供商级配置文件，并为特定模型添加更窄的覆盖。

### 从配置文件文件加载（Load profiles from config files）

对于无法直接使用 Python 的环境（例如 CI/CD 流水线、Kubernetes ConfigMap、平台 UI），配置文件也可以从 YAML 或 JSON 文件加载。在这些情况下，使用 `HarnessProfileConfig` 代替 `HarnessProfile`：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import HarnessProfileConfig, load_harness_profile

profile = load_harness_profile("path/to/profile.yaml")
```

YAML 结构：

```yaml theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
system_prompt_suffix: "Respond in under 100 words."
excluded_tools:
  - "execute"
excluded_middleware:
  - "SummarizationMiddleware"
general_purpose_subagent:
  enabled: false
```

## 提供商配置文件（Provider profiles）

提供商配置文件是 `HarnessProfile` 的更窄同伴。它们专门针对模型构造参数，不接触操控。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import ProviderProfile, register_provider_profile

register_provider_profile(
    "my_provider",
    ProviderProfile(
        init_kwargs={"temperature": 0},
    ),
)
```

## 自动发现（Auto-discovery）

配置文件通过标准的 Python 入口点自动发现。这适用于您自己的配置文件以及您安装的提供程序包（如 `langchain-anthropic`）中的配置文件。

**如何工作**：当 Deep Agents 导入 `deepagents.profiles` 时，它使用 `importlib.metadata.entry_points` 发现 `deepagents.harness_profiles` 和 `deepagents.provider_profiles` 组下的入口点。这与 `pytest` 插件、`mkdocs` 插件和类似可插拔框架使用的机制相同。

入口点定义在提供程序包的 `pyproject.toml` 中。您也可以在自己的用户代码中手动调用 `register_harness_profile()` 和 `register_provider_profile()`——所有三条路径都通过相同的加法注册汇聚，因此稍后在同一键下的注册会在早期注册之上叠加。

在发行版自己的 `pyproject.toml` 中声明入口点，放在相应的组下：

```toml theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
[project.entry-points."deepagents.harness_profiles"]
my_provider = "my_pkg.profiles:register_harness"

[project.entry-points."deepagents.provider_profiles"]
my_provider = "my_pkg.profiles:register_provider"
```

每个目标解析为一个零参数的可调用函数，在导入 `deepagents.profiles` 时执行注册：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from deepagents import (
    HarnessProfile,
    ProviderProfile,
    register_harness_profile,
    register_provider_profile,
)


def register_harness() -> None:
    register_harness_profile(
        "my_provider",
        HarnessProfile(system_prompt_suffix="Batch independent tool calls in parallel."),
    )


def register_provider() -> None:
    register_provider_profile(
        "my_provider",
        ProviderProfile(init_kwargs={"temperature": 0}),
    )
```
