# 人工介入（Human-in-the-loop）

> 在工具执行前要求人工审批

某些工具操作可能是敏感的，需要执行前获得人工审批。Deep Agents 通过 LangGraph 的中断（interrupt）能力支持人工介入工作流。您可以使用 `interrupt_on` 参数配置哪些工具需要审批。

## 基本配置（Basic configuration）

`interrupt_on` 参数接受一个字典，将工具名称映射到中断配置。每个工具可以配置为：

- **`True`**：启用中断，使用默认行为（允许审批、编辑、拒绝、回复）
- **`False`**：对此工具禁用中断
- **`{"allowed_decisions": [...]}`**：自定义配置，指定允许的决策类型

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from langchain.tools import tool
from deepagents import create_deep_agent
from langgraph.checkpoint.memory import MemorySaver


@tool
def remove_file(path: str) -> str:
    """Delete a file from the filesystem."""
    return f"Deleted {path}"


@tool
def fetch_file(path: str) -> str:
    """Read a file from the filesystem."""
    return f"Contents of {path}"


@tool
def notify_email(to: str, subject: str, body: str) -> str:
    """Send an email."""
    return f"Sent email to {to}"


# Checkpointer is REQUIRED for human-in-the-loop
checkpointer = MemorySaver()

agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    tools=[remove_file, fetch_file, notify_email],
    interrupt_on={
        "remove_file": True,  # Default: approve, edit, reject, respond
        "fetch_file": False,  # No interrupts needed
        "notify_email": {"allowed_decisions": ["approve", "reject"]},  # No editing
    },
    checkpointer=checkpointer,  # Required!
)
```

## 决策类型（Decision types）

`allowed_decisions` 列表控制人工在审查工具调用时可以执行的操作：

- **`"approve"`**：使用 Agent 提出的原始参数执行工具
- **`"edit"`**：在执行前修改工具参数
- **`"reject"`**：完全跳过执行此工具调用
- **`"respond"`**：直接将人工的消息作为工具结果返回，跳过执行——适用于"询问用户"类型的工具

您可以为每个工具自定义哪些决策可用：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
interrupt_on = {
    # Sensitive operations: allow all options
    "delete_file": {"allowed_decisions": ["approve", "edit", "reject"]},

    # Moderate risk: approval or rejection only
    "write_file": {"allowed_decisions": ["approve", "reject"]},

    # Must approve (no rejection allowed)
    "critical_operation": {"allowed_decisions": ["approve"]},
}
```

## 处理中断（Handle interrupts）

当中断被触发时，Agent 暂停执行并返回控制权。在结果中检查中断并相应处理。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from langchain_core.utils.uuid import uuid7
from langgraph.types import Command

# Create config with thread_id for state persistence
config = {"configurable": {"thread_id": str(uuid7())}}

# Invoke the agent
result = agent.invoke(
    {"messages": [{"role": "user", "content": "Delete the file temp.txt"}]},
    config=config,
    version="v2",  # [!code highlight]
)

# Check if execution was interrupted
if result.interrupts:  # [!code highlight]
    # Extract interrupt information
    interrupt_value = result.interrupts[0].value  # [!code highlight]
    action_requests = interrupt_value["action_requests"]  # [!code highlight]

    # Show user what the agent wants to do
    for action in action_requests:
        print(f"Tool: {action['tool_name']}")
        print(f"Args: {action['args']}")

    # Collect user decisions (in a real app, this would be UI)
    decisions = []
    for action in action_requests:
        decision = input(f"Approve {action['tool_name']}? (approve/reject): ")
        decisions.append(decision)

    # Resume execution with decisions
    result = agent.invoke(
        Command(resume={"decisions": decisions}),
        config=config,
        version="v2",
    )

# Process final result
print(result)
```

## 多个工具调用（Multiple tool calls）

当 Agent 在一次响应中调用多个工具时，所有工具都会在继续之前请求审批。您必须按照 `action_requests` 的顺序为每个工具提供决策：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# Two tools need approval
action_requests = interrupt_value["action_requests"]
# action_requests = [
#     {"tool_name": "remove_file", "args": {"path": "temp.txt"}},
#     {"tool_name": "notify_email", "args": {"to": "admin@example.com", ...}},
# ]

# Provide decisions in the same order as action_requests
result = agent.invoke(
    Command(resume={"decisions": ["approve", "reject"]}),
    config=config,
    version="v2",
)
```

## 编辑工具参数（Edit tool arguments）

当 `"edit"` 是允许的决策时，用户可以修改工具参数。在 `decisions` 列表中为每个动作返回一个决策字符串或一个带有 `type` 和 `args` 的字典：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# Original args from the agent
action_requests = [
    {
        "tool_name": "notify_email",
        "args": {"to": "admin@example.com", "subject": "Alert", "body": "..."},
    },
]

# User decides to edit the recipient
result = agent.invoke(
    Command(
        resume={
            "decisions": [
                {
                    "type": "edit",
                    "args": {"to": "team@example.com", "subject": "Alert", "body": "..."},
                },
            ]
        }
    ),
    config=config,
    version="v2",
)
```

## 子 Agent 中断（Subagent interrupts）

当中断发生在子 Agent 中时，数据流取决于中断发生在工具层级还是工具调用内部。

### 工具调用的中断（Interrupts on tool calls）

子 Agent 默认继承父级的中断配置。您可以为特定子 Agent 指定不同的 `interrupt_on` 配置来覆盖它：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
agent = create_deep_agent(
    model="google_genai:gemini-3.1-pro-preview",
    subagents=[
        SubAgent(
            name="research_assistant",
            description="Research and data analysis",
            interrupt_on={
                # Override: require approval for reads in this subagent
                "read_file": {"allowed_decisions": ["approve", "reject"]},
            },
        ),
    ],
)
```

### 工具调用内部的中断（Interrupts within tool calls）

工具本身可以使用 `interrupt()` 暂停执行。这在工具需要执行过程中的人工输入时很有用：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
from langgraph.types import Command, interrupt

@tool
def draft_contract(terms: str) -> str:
    """Draft a contract based on the specified terms."""
    # interrupt() pauses execution and returns the value passed to Command
    result = interrupt(
        Command(
            resume={
                "action_requests": [
                    {
                        "tool_name": "draft_contract",
                        "args": {"terms": terms},
                        "type": "human_review",
                    }
                ]
            }
        )
    )

    decisions = result.get("decisions", [])
    if not decisions or decisions[0] == "reject":
        return "Contract draft was rejected."

    return f"Contract drafted with terms: {terms}"
```

然后处理中断：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# Check for interrupt
if result.interrupts:
    interrupt_value = result.interrupts[0].value

    if interrupt_value.get("type") == "human_review":
        # Show contract terms to user for review
        print(f"Contract terms: {interrupt_value['action_requests'][0]['args']['terms']}")

        # Resume with approval
        result = agent.invoke(
            Command(resume={"decisions": ["approve"]}),
            config=config,
            version="v2",
        )
```

## 最佳实践（Best practices）

### 始终使用检查点器（Always use a checkpointer）

人工介入需要一个检查点器（checkpointer）来保存 Agent 在中断点的状态。没有它，Agent 在恢复时无法继续。使用 `MemorySaver` 进行本地开发，部署到 LangSmith 时使用平台的检查点器。

### 使用相同的线程 ID（Use the same thread ID）

首次调用和恢复调用必须使用相同的 `thread_id`。检查点器使用线程 ID 来查找保存的状态：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# First call
config = {"configurable": {"thread_id": "my-thread-123"}}
result = agent.invoke({...}, config=config, version="v2")

# Resume (use same config)
result = agent.invoke(
    Command(resume={"decisions": ["approve"]}),
    config=config,
    version="v2",
)
```

### 使决策顺序与操作匹配（Match decision order to actions）

当 Agent 同时请求多个审批时，您的决策列表必须与 `action_requests` 的顺序完全匹配。如果顺序不匹配，工具将使用错误的参数执行：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# Create one decision per action, in order
decisions = []
for action in action_requests:
    if action["tool_name"] == "delete_file":
        decisions.append("approve")
    elif action["tool_name"] == "send_email":
        decisions.append({
            "type": "edit",
            "args": {**action["args"], "to": "safe@example.com"},
        })
    else:
        decisions.append("reject")

result = agent.invoke(
    Command(resume={"decisions": decisions}),
    config=config,
    version="v2",
)
```

### 根据风险定制配置（Tailor configurations by risk）

根据风险级别为不同工具配置不同的配置：

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
interrupt_on = {
    # High risk: full control (approve, edit, reject)
    "delete_file": {"allowed_decisions": ["approve", "edit", "reject"]},
    "send_email": {"allowed_decisions": ["approve", "edit", "reject"]},

    # Medium risk: no editing allowed
    "write_file": {"allowed_decisions": ["approve", "reject"]},

    # Low risk: no interrupts
    "read_file": False,
    "list_files": False,
}
```
