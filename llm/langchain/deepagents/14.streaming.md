# 流式传输（Streaming）

> 从 Deep Agents 流式传输 LLM token、子 Agent 进度和工具调用

本页面涵盖 Deep Agents 特有的流式传输问题——最重要的是，通过 `stream.subagents` 从委托的子 Agent 进行流式传输。关于通用 Agent 流式传输（`stream.messages`、`stream.values`、工具调用、自定义更新），请参见 [LangChain Event Streaming](/oss/python/langchain/event-streaming)。

对于新应用，我们推荐使用事件流（Event streaming）——Deep Agents v0.6 中引入的类型化投影 API。事件流为每个投影（子 Agent、消息、工具调用、值）提供独立的迭代器，因此您可以独立消费它们，而不必在 `stream_mode` 块上进行分支。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
with agent.stream_events(
    {"messages": [{"role": "user", "content": "Research X"}]},
    version="v3",
) as stream:
    for subagent in stream.subagents:
        match subagent.status:
            case "started":
                print(f"[started] {subagent.name}")
            case "completed":
                print(f"[completed] {subagent.name}: {subagent.output}")

    for message in stream.messages:
        print(message.pretty_print())
```

<Note>
  需要 LangGraph >= 1.1。
</Note>

## 启用子图流式传输（Enable subgraph streaming）

Deep Agents 在 [LangGraph 的 `subgraphs` 流式传输功能](https://langchain-ai.github.io/langgraph/how-tos/stream-subgraphs/)之上构建。要获取子 Agent 生成的事件，请在调用 `agent.stream()` 时设置 `subgraphs=True`。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
# subgraphs=True gives namespace + event type info
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "Hello"}]},
    stream_mode=["updates", "messages"],
    subgraphs=True,
):
    mode = chunk[0]  # "updates" | "messages"
    namespace = chunk[1]  # () for main, ("tools:<id>",) for subagent
    data = chunk[2]  # the actual payload
```

### 流式传输 v2 格式

Deep Agents 支持 LangGraph v2 流式传输格式，该格式使用命名元组，并通过 `version="v2"` 参数启用。

**输出**

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
{
    "type": "updates" | "messages" | "custom",
    "ns":   (),                          # main agent
            ("tools:<subagent_id>",),    # subagent
    "data": { ... }                      # the payload
}
```

<CodeGroup>

  for chunk in agent.stream(

## 命名空间（Namespaces）

当 `subgraphs=True` 时，每个流式块包含一个指示来源的 `namespace`。主 Agent 使用空元组 `()`。子 Agent 使用包含工具调用 ID 的元组，如 `("tools:<tool_call_id>",)`。

```python
# Check if this event came from a subagent
namespace = chunk.get("ns") or chunk[1]  # v2 or v1 tuple
is_subagent = bool(namespace)
```

```python
# Extract the tool call ID from the namespace
if namespace:
    tool_call_id = namespace[0].split(":")[1]
    print(f"Event from subagent tool call: {tool_call_id}")
```

这种命名空间方案允许您以编程方式将事件路由到适当的 UI 组件或日志通道。

## 子 Agent 进度（Subagent progress）

使用 `stream_mode="updates"` 并在 Agent 处理时观察状态变化：

```python
for mode, namespace, data in agent.stream(
    {"messages": [{"role": "user", "content": "Research X and code Y"}]},
    stream_mode=["updates"],
    subgraphs=True,
):
    if mode == "updates":
        if not namespace:
            # Main agent updates (empty namespace)
            if "__end__" not in data:
                print(f"[main] {list(data.keys())}")
        else:
            # Subagent results returned to main agent
            agent_name = namespace[0]
            print(f"[{agent_name}] completed")
            print(data)
```

要获取所有子 Agent 更新（不包括最终聚合结果），请检查命名空间是否存在并且数据不包含 `__end__`：

```python
for mode, namespace, data in agent.stream(..., stream_mode=["updates"], subgraphs=True):
    if mode == "updates" and namespace and "__end__" not in data:
        # Subagent updates (non-empty namespace)
        print(f"[subagent {namespace[0]}] progress update: {data}")
```

## LLM token（LLM tokens）

要获取每个 token 的流式传输，使用 `stream_mode="messages"`。这为每个 token 发出一个事件，包含内容、角色和工具调用信息。

```python
for mode, namespace, data in agent.stream(
    {"messages": [{"role": "user", "content": "Write a poem"}]},
    stream_mode=["messages"],
    subgraphs=True,
):
    if mode != "messages":
        continue

    chunk = data[0]
    # Check if this event came from a subagent (namespace contains "tools:")
    is_subagent = bool(namespace)

    if hasattr(chunk, "content") and chunk.content:
        if is_subagent:
            # Token from a subagent
            print(f"[subagent] {chunk.content}", end="", flush=True)
        else:
            # Token from the main agent
            print(f"[main] {chunk.content}", end="", flush=True)
```
