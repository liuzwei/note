# 事件流（Event streaming）

> 从 Deep Agents 流式传输子 Agent、消息、工具调用和最终输出

本页面涵盖 Deep Agents 特有的事件流问题——最重要的是，通过 `stream.subagents` 从委托的子 Agent 进行流式传输。关于通用 Agent 流式传输（`stream.messages`、`stream.values`、工具调用、自定义更新），请参见 [LangChain Event Streaming](/oss/python/langchain/event-streaming)。

## 流式传输子 Agent（Stream subagents）

Deep Agents 在 LangGraph 流式传输之上添加了子 Agent 投影（subagent projection）。当您希望每个委托的 `task` 调用都有一个独立的流句柄时，使用 `stream.subagents`。该投影是轻量级的：它首先发现子 Agent 任务，只有在您访问子 Agent 句柄上的特定投影时，消息、工具调用和值流才会被打开。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
stream = agent.stream_events({
    "messages": [{"role": "user", "content": "Write me a haiku about the sea"}],
}, version="v3")

for subagent in stream.subagents:
    print(subagent.name, subagent.path, subagent.status)

    for message in subagent.messages:
        print(message.text)
```

## 子 Agent 流字段（Subagent stream fields）

每个子 Agent 流公开与父运行相同类型的投影，例如消息、工具调用、嵌套子 Agent 和最终输出。关于通用父运行流式传输模型，请参见 [LangChain Event Streaming](/oss/python/langchain/event-streaming)。

Python 使用 snake_case 投影名称，如 `tool_calls`。每个子 Agent 流可以公开 `.messages`、`.tool_calls`、`.values`、`.subagents` 和 `.output`。

| 字段 | 描述 |
| ------------ | ---------------------------------------------------------------------------- |
| `name` | 子 Agent 名称。 |
| `messages` | 子 Agent 发出的消息。 |
| `subagents` | 嵌套的子 Agent 调用。 |
| `output` | 子 Agent 的最终状态，或委托任务的完成信号。 |
| `path` | 子 Agent 流的命名空间路径。 |
| `status` | 生命周期状态，如 `started`、`completed`、`failed` 或 `interrupted`。 |
| `tool_calls` | 限定到子 Agent 的工具调用。 |

## 使用事件流（Event streaming API）

Deep Agents v0.6 引入了 `stream_events` API，这是一个类型化的投影 API，提供每个流模式的独立迭代器。对于面向用户的 UI，这是推荐的方法，因为它允许您独立消费子 Agent 事件、消息和工具调用，而无需处理原始 `stream_mode` 块。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
stream = agent.stream_events({
    "messages": [{"role": "user", "content": "Research X and write a summary."}],
}, version="v3")

for subagent in stream.subagents:
    if subagent.status == "started":
        print(f"Delegated to: {subagent.name}")

for message in stream.messages:
    if message.type == "ai":
        print(message.text)

for tool_call in stream.tool_calls:
    print(f"Tool used: {tool_call.name}")
```

### 推荐用法（Recommended usage）

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
with agent.stream_events({"messages": [{"role": "user", "content": "Research"}]}, version="v3") as stream:
    for subagent in stream.subagents:
        match subagent.status:
            case "started":
                ui.add_card(subagent.name, subagent.path)
            case "completed":
                ui.update_card(subagent.path, status="done", output=subagent.output)
            case "interrupted":
                ui.update_card(subagent.path, status="blocked")
```

## 流式传输 v2（Streaming v2）

在某些情况下，您可能在使用 `stream_events` 的事件流 API 之前处理原始的 `stream_mode` 块。`v2` `stream_mode` 格式在父运行和子 Agent 运行之间提供统一的契约，与底层图执行保持一致。

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "Research X"}]},
    stream_mode=["updates", "messages", "custom"],
    subgraphs=True,
    version="v2",
):
    print(chunk["type"])  # "updates" | "messages" | "custom"
    print(chunk["ns"])    # () main, ("tools:<id>",) subagent
    print(chunk["data"])  # payload
```

### 过滤子 Agent token（Filtering subagent tokens）

```python theme={"theme":{"light":"catppuccin-latte","dark":"catppuccin-mocha"}}
for chunk in agent.stream(...):
    if chunk["type"] != "messages":
        continue

    payload = chunk["params"]["data"][0]
    if not isinstance(payload, dict):
        continue
    if payload.get("event") != "content-block-delta":
        continue

    block = payload.get("delta") or {}
    if block.get("type") == "text-delta":
        source = "subagent" if chunk["params"]["namespace"] else "coordinator"
        print(f"[{source}] {block['text']}")
```

## 子 Agent 与子图（Subagents versus subgraphs）

`stream.subgraphs` 显示图执行结构。`stream.subagents` 显示产品级别的 Deep Agents 任务委派。在面向用户的 UI 中使用 `stream.subagents`，因为它隐藏了内部图节点，直接暴露子 Agent 概念。
