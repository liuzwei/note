### 论文信息

* 标题：GQA: Training Generalized Multi-Query Transformer Models from Multi-Head Checkpoints
* 作者：Joshua Ainslie、James Lee-Thorp、Michiel de Jong、Yury Zemlyanskiy、Federico Lebrón、Sumit Sanghai
* 机构：Google Research
* arXiv：2305.13245v3 [cs.CL]
* 原文链接：<https://arxiv.org/abs/2305.13245>
* PDF：<https://arxiv.org/pdf/2305.13245.pdf>

---

### 解释：这篇论文到底在解决什么问题

这篇论文关注的是 **大语言模型在解码阶段为什么这么慢**。

在自回归生成里，每生成一个 token，模型都要反复读取：

* 解码器参数
* 历史 token 的 key/value cache

随着上下文变长，`KV cache` 的带宽开销会越来越明显，推理速度就会被拖慢。

已有一个经典办法叫 **MQA（Multi-Query Attention）**：

* query 头仍然很多
* 但所有头共享同一组 key/value

这样可以显著减少 KV cache 的读取量，所以推理更快。  
问题是：**MQA 往往会带来质量下降**，而且如果为了推理速度重新训练一套模型，成本又太高。

这篇论文要解决的核心矛盾就是：

**怎样在不付出完整重训代价的前提下，把多头注意力模型改造成推理更快、但质量尽量不掉的结构？**

作者给了两个答案：

1. **Uptraining**：把已有的 `MHA` 检查点转换后，再用原始预训练算力的 `5%` 继续训练一小段
2. **GQA（Grouped-Query Attention）**：在 `MHA` 和 `MQA` 之间引入一个折中结构

---

### 一句话总结整篇论文

**GQA 证明了：可以把已经训练好的多头 Transformer 检查点，以很低额外成本升级成一种更适合推理的注意力结构，并在几乎接近 MQA 速度的同时，保留接近 MHA 的质量。**

---

### 摘要翻译整理

* MQA 只使用一个 key-value head，能显著加快 decoder 推理。
* 但 MQA 可能带来质量下降，而且很多时候不值得为更快推理单独训练一套模型。
* 本文首先提出一种 **uptraining 配方**，只用原始预训练计算量的 `5%`，就能把已有的多头语言模型检查点转成 MQA 模型。
* 然后作者提出 **GQA**，它是 MQA 的推广形式：key/value head 的数量不是 `1`，也不是和 query head 数量一样多，而是取一个中间值。
* 实验表明：**经过 uptraining 的 GQA，在质量上接近 MHA，在速度上接近 MQA。**

---

### 论文的核心贡献

1. 提出一种从 **MHA 检查点升级到 MQA/GQA** 的低成本训练方案
2. 提出 **Grouped-Query Attention**，作为 `MHA <-> MQA` 的连续折中
3. 说明 **分组后的 key/value 共享** 可以在速度和效果之间提供更好的 Pareto trade-off
4. 在 T5 Large / T5 XXL 上验证：只做少量继续预训练，也能把结构改造成功

---

### 先理解三个概念：MHA、MQA、GQA

#### MHA：Multi-Head Attention

标准多头注意力里，每个 query head 都有自己独立的 key 和 value head。  
优点是表达能力强，缺点是推理时需要维护和读取大量 KV cache。

#### MQA：Multi-Query Attention

MQA 保留多个 query head，但 **所有 query 头共享同一个 key head 和同一个 value head**。

优点：

* KV cache 体积更小
* 推理时内存带宽开销显著下降
* 解码更快

缺点：

* 表达能力被压缩
* 质量可能下降
* 训练也可能更不稳定

#### GQA：Grouped-Query Attention

GQA 的做法是把 query heads 分成 `G` 组：

* 每组 query 共享一组 key head
* 每组 query 共享一组 value head

于是：

* `GQA-1 = MQA`
* `GQA-H = MHA`，其中 `H` 是 query head 总数

所以 GQA 本质上就是：

**不是把所有 query 都压到一组 KV 上，而是只做“部分共享”。**

这让它天然处在 MHA 和 MQA 的中间地带。

---

### 1. 引言翻译整理

作者指出，大模型自回归推理的主要瓶颈之一，是 **每一步都要把 attention 的 keys 和 values 从内存中加载出来**。

MQA 可以显著降低这一开销，因为它只保留一组 key/value heads。  
但 MQA 的问题也很明显：

* 容量下降
* 质量下降
* 可能不稳定
* 很多现有开源或已有模型并不是按 MQA 训练的

例如论文里提到：

* `PaLM` 使用了 MQA
* 但 `T5`、`LLaMA` 这类模型并没有

因此作者的目标不是“从零训练一个新结构”，而是：

**能不能把已经训练好的 MHA 模型，低成本地改造成更快的推理模型？**

---

### 2. 方法翻译整理

#### 2.1 Uptraining：从多头检查点升级

把一个 MHA 模型变成 MQA/GQA，作者分两步做：

1. **先改 checkpoint 结构**
2. **再继续预训练一小段，让模型适应新结构**

对于从 `MHA -> MQA` 的转换，作者不是直接随机初始化新的 key/value heads，也不是简单保留某一个 head，而是：

**把所有原始 key/value projection matrix 做 mean pooling，合并成新的共享 key/value projection。**

论文实验说明，这种方式效果最好，因为它保留了更多原始模型信息。

转换之后，再沿用原模型相同的预训练 recipe 和数据，继续训练一小部分步数。  
作者把这部分额外训练量记作 `α`，主实验用的是：

* `α = 0.05`

也就是只用原始预训练计算量的 `5%` 做继续训练。

#### 2.2 GQA：分组共享 key/value

GQA 把 query heads 划分成 `G` 个组，每组共享一套 key/value。

如果原来有 `H` 个 heads，那么：

* `G = 1` 时，所有 heads 共用一个 KV，对应 MQA
* `G = H` 时，每个 head 自己有 KV，对应 MHA
* `1 < G < H` 时，就是论文提出的 GQA

从 checkpoint 转成 GQA 时，作者会对 **组内原始 heads 的 key/value 参数做均值池化**，得到该组共享的 key/value head。

#### 为什么 GQA 有机会成为更好的折中

作者给出的直觉是：

* MHA 质量高，但 KV cache 最大
* MQA 速度快，但压缩过于激进
* GQA 只做部分共享，因此既减少带宽，又尽量保留容量

另外，模型越大，head 数通常越多。  
这意味着从 MHA 直接退化到 MQA，有时会是一次非常剧烈的容量削减；而 GQA 可以把这种削减做得更平滑。

论文还强调：

* GQA 只应用在 **decoder self-attention** 和 **cross-attention**
* **encoder self-attention 不改**

原因是 encoder 计算是并行的，通常不是推理带宽瓶颈。

### 3. 实验设置

#### 模型与实现

* 基于 `T5.1.1`
* 框架：`JAX`、`Flax`、`Flaxformer`
* 主要考察：
  * `T5 Large + MHA`
  * `T5 XXL + MHA`
  * `T5 XXL + MQA`（由 MHA checkpoint uptrain 得到）
  * `T5 XXL + GQA-8`（由 MHA checkpoint uptrain 得到）

#### 训练设定

* 优化器：`Adafactor`
* uptraining 初始化：来自公开的 `T5.1.1 checkpoints`
* 继续预训练比例：主实验使用 `α = 0.05`
* 论文给出的成本：`α = 0.05` 时，约为 `600 TPUv3 chip-days`

#### 评测任务

作者评测了多种生成任务：

* 摘要：`CNN/DailyMail`
* 长文摘要：`arXiv`、`PubMed`、`MediaSum`、`MultiNews`
* 翻译：`WMT 2014 English-to-German`
* 问答：`TriviaQA`

作者没有测 GLUE 这类分类任务，因为这篇论文关注的是 **自回归推理速度**，而不是通用编码器任务。

### 4. 主实验结果

下表是论文 Table 1 的关键结果整理。`Tinfer` 表示每个样本在 TPUv4 上的平均推理时间。

| 模型 | Tinfer（秒） | 平均表现 | CNN/DM R1 | arXiv R1 | PubMed R1 | MediaSum R1 | MultiNews R1 | WMT BLEU | TriviaQA F1 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| MHA-Large | 0.37 | 46.0 | 42.9 | 44.6 | 46.2 | 35.5 | 46.6 | 27.7 | 78.2 |
| MHA-XXL | 1.51 | 47.2 | 43.8 | 45.6 | 47.5 | 36.4 | 46.9 | 28.4 | 81.9 |
| MQA-XXL | 0.24 | 46.6 | 43.0 | 45.0 | 46.9 | 36.1 | 46.5 | 28.5 | 81.3 |
| GQA-8-XXL | 0.28 | 47.1 | 43.5 | 45.4 | 47.7 | 36.3 | 47.2 | 28.4 | 81.6 |

#### 如何理解这张表

先看速度：

* `MHA-XXL` 最慢，`1.51s`
* `MQA-XXL` 最快，`0.24s`
* `GQA-8-XXL` 是 `0.28s`，几乎和 MQA 一样快

再看质量：

* `MQA-XXL` 相比 `MHA-XXL` 有轻微退化
* `GQA-8-XXL` 基本追到了 `MHA-XXL`
* 在某些任务上，`GQA-8-XXL` 甚至略好于 `MHA-XXL`

所以这篇论文最重要的结论就是：

**GQA 在几乎保留 MQA 速度优势的同时，把质量拉回到了接近 MHA 的水平。**

论文图 3 也表达了同样结论：

* 单看 MQA，已经比 `MHA-Large` 更快且质量更高
* 再换成 GQA，则在几乎不增加多少推理开销的情况下，进一步提高了效果

### 5. 消融实验

#### 5.1 Checkpoint 转换方式

作者比较了三种把 MHA checkpoint 改成 MQA 的方法：

* `Mean`：对 key/value heads 做均值池化
* `First`：只取第一个 head
* `Random`：随机初始化新 heads

结果是：

**Mean pooling 最好，First 次之，Random 最差。**

这很好理解，因为：

* mean pooling 保留了更多原始预训练信息
* 只取一个 head 会丢掉其他 heads 的知识
* 随机初始化几乎等于强行让模型重新学

#### 5.2 Uptraining 步数够不够

作者比较了不同 `α` 值下的效果，结论很明确：

* `GQA` 在刚转换后就已经有比较可用的表现
* `MQA` 更依赖后续 uptraining
* `5%` 的继续预训练已经能带来明显收益
* 从 `5%` 增加到 `10%`，收益开始递减

也就是说：

**5% 算力是一个很有性价比的点。**

#### 5.3 分组数怎么选

作者还研究了 GQA 的 group 数量对速度的影响。

结论是：

* 从 `1` 组（MQA）增加到 `8` 组，推理速度只会有相对温和的变慢
* 当 group 数继续增多、逐渐接近 MHA 时，速度成本开始明显上升

因此论文最终选择：

* `GQA-8`

作为比较合适的中间点。

### 6. 相关工作翻译整理

这篇论文放在更大的背景里，属于一类共同目标明确的工作：

**通过减少推理时的内存带宽压力，提升大模型生成速度。**

作者提到的相关方向包括：

* `MQA`：直接减少 KV heads 数量
* `FlashAttention`：优化 attention 计算和显存访问方式
* `Quantization`：降低权重和激活精度，减少读写量
* `Distillation`：用更小的模型逼近大模型
* `Layer-sparse cross-attention`：减少长输入情况下 cross-attention 的成本
* `Speculative Sampling`：用小模型先提议 token，再由大模型并行验证

本文的独特点在于：

**它不是单纯提出一种新注意力结构，而是把“已有 MHA 模型如何低成本迁移到更快结构”这个工程问题直接做通了。**

---

### 7. 结论翻译整理

作者最后的结论可以浓缩为三句话：

1. 语言模型推理昂贵，一个关键原因是读取 keys / values 的带宽开销
2. MQA 能缓解这个问题，但会损失部分质量
3. 通过少量 uptraining 加上 GQA 结构，可以在速度和质量之间得到更好的折中

换句话说，这篇论文真正提供的是一种很实用的工程路径：

**不是推倒重练，而是把已经训练好的强模型，升级成更适合部署的模型。**

---

### 8. 局限性

论文也明确承认了几个局限：

1. 它主要优化的是 **长序列生成时的带宽瓶颈**，而长文本质量本身就不容易准确评估
2. 摘要任务主要依赖 `ROUGE`，这个指标并不完美
3. 受算力限制，作者没有把 `XXL GQA` 和“从零训练的同结构模型”做严格对比
4. 论文实验只覆盖了 **encoder-decoder** 架构，没有系统验证 **decoder-only** 大模型

作者还提出一个很重要的推测：

**在 decoder-only 模型里，GQA 相对 MQA 的优势可能更明显。**

后来的很多大模型实践，基本验证了这个判断。

---

### 9. 这篇论文为什么重要

这篇论文的重要性不只在于提出了 GQA 这个名字，更在于它把今天大模型部署里一个非常常见的选择解释清楚了：

* 为什么很多模型不再坚持纯 MHA
* 为什么又不完全退化成 MQA
* 为什么 `GQA` 经常成为工程上的默认折中方案

如果你现在看 LLM 推理优化，会发现很多模型在注意力结构上都采用了类似思想。  
从这个意义上说，这篇论文是 **KV cache 时代推理优化路线** 里非常关键的一篇。

---

### 10. 我的理解：可以把它看成一次“结构压缩后的短暂恢复训练”

如果用更直观的话来理解，这篇论文做的事情像是：

1. 先把一个表达能力更强、但更慢的 MHA 模型压缩成更省带宽的结构
2. 再给它一小段恢复训练，让它重新适应这种压缩后的参数组织方式
3. 最后找到一个压缩幅度不那么激进的中间点，也就是 GQA

所以 GQA 的价值并不只是“多了几个 group”这么简单，而是：

**它说明结构改造不一定要从头训练，已有 checkpoint 本身就是很有价值的迁移起点。**
